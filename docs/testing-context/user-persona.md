# User Persona

## 主要ユーザー種別

| ユーザー | 主な操作 | 重視する体験 |
|---------|---------|-------------|
| タスク作成者 | タスク作成、ステータス遷移、承認申請 | 自分のタスクが正しく進行すること |
| チームリーダー | タスク承認・却下、進捗確認 | 承認フローが滞りなく動くこと |
| 管理者 | 全タスクの管理、ユーザー管理 | 全体の状態把握と緊急対応ができること |
| 閲覧者 | タスク一覧の閲覧 | 見るだけで誤操作しないこと |

## 権限ロール

| ロール | タスク作成 | ステータス遷移 | 承認・却下 | 備考 |
|-------|-----------|--------------|-----------|------|
| `viewer` | 不可 | 不可 | 不可 | 読み取り専用 |
| `editor` | 可 | 可 | 不可 | 一般メンバー |
| `lead` | 可 | 可 | 可 | チームリーダー |
| `admin` | 可 | 可 | 可 | システム管理者 |

ロールは HTTP ヘッダー `X-User-Role` で伝搬する。認証基盤との連携は現時点で未整理。

## 誤操作しやすいユーザー像

- **editor が承認しようとする**: 権限不足で 403 になるが、UI がそれを事前に防止しているか要確認
- **viewer がステータス遷移ボタンを見てしまう**: ボタンの表示/非表示がロールに連動しているか
- **lead が自分の担当タスクを自己承認する**: 自己承認を許可するかどうかは現時点で未整理（現在は許可されている）

## どの操作が誰にとって重要か

- **承認フロー**: lead / admin にとって最重要。誤承認・承認漏れはワークフロー停滞に直結
- **タスク作成**: editor にとっての日常操作。バリデーションエラーが分かりやすいか
- **一覧表示**: 全ロールが依存。表示が壊れると全員に影響
